# 阶段性整改记录

## 2026-04-26

本轮整改目标：先不进行 MCP 独立服务器部署，优先完善功能闭环、多角色 Agent 协同和测试材料。

### 已完成

1. 多角色 Agent 协同骨架
   - `CoordinatorAgent`：负责任务规划。
   - `TaskAgent`：负责待办提取、任务创建、任务查询。
   - `WriterAgent`：负责通知和邮件草稿生成。
   - `DocumentAgent`：负责文档摘要和文本提取。
   - `SafetyAgent`：负责确认前后的执行边界。

2. 安全确认流程修正
   - 修正前：`create_tasks` 在用户确认前执行。
   - 修正后：确认前只执行 `extract_todos` 和 `generate_notice`；用户回复“确认”后再执行 `create_tasks`。

3. AI 工具结构化抽取预留
   - 新增 `LLMService.extract_todos_structured()`（AI 工具结构化抽取接口）。
   - 配置 `DEEPSEEK_API_KEY` 后可优先使用 AI 工具结构化抽取。
   - 未配置模型 Key 时自动回退到规则抽取工具。

4. 文档处理最小闭环
   - 支持 `.docx`、`.pdf`、`.xlsx`、`.txt` 文本提取。
   - `summarize_document` 返回摘要、文本片段、字符数和来源路径。

5. 自动化测试补充
   - Planner 测试。
   - 确认流程测试。
   - 文档解析测试。
   - 任务创建测试。

## 2026-04-26 二次优化

本轮整改目标：在已有邮件、Office 文档能力基础上，继续补强安全确认、测试稳定性和文档工具细节。

### 已完成

1. 邮件发送安全边界
   - 直接发送邮件请求不再立即触发 SMTP 发送。
   - 系统先生成邮件草稿，并要求用户回复“确认”后再执行发送。
   - 确认阶段会保留原始请求内容，避免用户只回复“确认”后丢失收件人邮箱地址。

2. 文档与表格工具优化
   - 测试环境允许使用 pytest 临时目录，生产环境仍限制输出在项目目录内。
   - 修复 Excel 多 sheet 创建时只复用第一个工作表的问题。
   - 明确移除旧版 `.xls` 的直接读取声明，避免 openpyxl 无法处理旧格式造成误导。

3. 邮件测试隔离
   - 增加 pytest 测试运行配置。
   - 测试环境启用邮件 dry-run，防止配置了真实 SMTP 后误发测试邮件。
   - 为邮件确认流程新增回归测试。

4. 依赖清单补齐
   - 补充 `reportlab`，用于 PDF 生成。
   - 补充 `pytest-asyncio`，便于标准异步测试环境运行。
