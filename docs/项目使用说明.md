# 项目使用说明

## 概述

Office Agent 是一个基于 MCP (Model Context Protocol) 的多平台办公协同智能体系统，支持通过 API 和 MCP 两种方式调用，实现办公任务的自动化处理，包括会议纪要整理、待办任务提取、通知生成、邮件发送等功能。

## 环境要求

- Python 3.8+
- 支持的操作系统：macOS, Linux, Windows

## 安装与配置

### 1. 克隆项目

```bash
git clone https://github.com/Na0333/office-agent.git
cd office-agent
```

### 2. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate  # Linux/macOS
# 或在 Windows: .venv\Scripts\activate
```

### 3. 安装依赖

```bash
pip install -r requirements.txt
```

### 4. 配置环境变量

复制示例配置文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件，配置以下关键参数：

```env
# AI 工具配置
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=your-api-key
LLM_MODEL=deepseek-chat

# 数据库配置
DATABASE_PATH=data/office_agent.db

# MCP 配置
MCP_TRANSPORT=stdio  # 或 streamable-http
MCP_HOST=0.0.0.0
MCP_PORT=8010

# 邮件配置（可选）
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USERNAME=your-email@qq.com
SMTP_PASSWORD=your-auth-code
SMTP_USE_TLS=true
SMTP_USE_STARTTLS=false
```

#### 邮件配置说明

- **QQ邮箱**：SMTP_HOST=smtp.qq.com, PORT=465, USE_TLS=true
- **Gmail**：SMTP_HOST=smtp.gmail.com, PORT=465, USE_TLS=true
- **Outlook**：SMTP_HOST=smtp-mail.outlook.com, PORT=587, USE_TLS=false, USE_STARTTLS=true

获取授权码：
- QQ邮箱：登录网页版 -> 设置 -> 账户 -> POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务 -> 开启POP3/SMTP服务
- Gmail：账户设置 -> 安全性 -> 两步验证 -> 应用密码
- Outlook：账户设置 -> 安全性 -> 更多安全性选项 -> 创建应用密码

### 5. 初始化数据库

```bash
python -c "from app.database.db import init_db; init_db()"
```

## 启动服务

### API 模式

```bash
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### MCP stdio 模式

```bash
PYTHONPATH=/path/to/office-agent \
DATABASE_PATH=/path/to/office-agent/data/office_agent.db \
MCP_TRANSPORT=stdio \
python -m app.mcp_server.server
```

### MCP Streamable HTTP 模式

```bash
PYTHONPATH=/path/to/office-agent \
DATABASE_PATH=/path/to/office-agent/data/office_agent.db \
MCP_TRANSPORT=streamable-http \
MCP_HOST=0.0.0.0 \
MCP_PORT=8010 \
python -m app.mcp_server.server
```

## API 调用

### 健康检查

```bash
curl http://127.0.0.1:8000/health
```

响应：`{"status": "ok"}`

### 获取能力清单

```bash
curl http://127.0.0.1:8000/capabilities
```

### 聊天接口

```bash
curl -X POST http://127.0.0.1:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "platform": "web",
    "user_id": "u001",
    "session_id": "s001",
    "content": "请把这段会议纪要整理成待办，并生成通知：张三周五前完成需求文档，李四下周一提交测试计划。"
  }'
```

## MCP 调用

### 工具列表

系统提供以下 MCP 工具：

1. `extract_todos_tool` - 从文本中提取待办事项
2. `create_tasks_tool` - 创建任务
3. `query_task_status_tool` - 查询任务状态
4. `generate_notice_tool` - 生成通知草稿
5. `generate_email_tool` - 生成邮件草稿
6. `summarize_document_tool` - 文档摘要
7. `send_email_tool` - 发送邮件
8. `office_agent_chat_tool` - 完整办公流程

### 示例调用

在支持 MCP 的客户端中，可以调用工具：

```json
{
  "method": "tools/call",
  "params": {
    "name": "extract_todos_tool",
    "arguments": {
      "content": "会议纪要：张三负责需求文档，周五前完成。"
    }
  }
}
```

## AstrBot 插件集成

1. 将 `astrbot_plugin_office_agent` 目录复制到 AstrBot 插件目录
2. 重启 AstrBot
3. 在聊天中发送消息，插件会自动转发到 Office Agent API

## 故障排除

### 常见问题

1. **AI 工具调用失败**：检查 API_KEY 和 BASE_URL
2. **数据库错误**：确保 DATABASE_PATH 路径存在且可写
3. **邮件发送失败**：验证 SMTP 配置和授权码
4. **MCP 连接失败**：检查端口和传输模式

### 日志查看

服务运行时会输出详细日志，帮助诊断问题。
