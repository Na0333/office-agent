## 1. 项目简介

Office Agent 是一个基于 MCP (Model Context Protocol) 协议的多平台办公协同智能体。

### 核心功能

| 功能 | 说明 |
|------|------|
| 会议纪要处理 | 自动解析会议记录，提取待办事项和负责人 |
| 文档处理 | 读取/创建/转换 Word、Excel、PDF、文本文件 |
| 邮件发送 | 支持 QQ、QQ企业邮箱、Gmail、Outlook 等 SMTP |
| 跨平台通知 | 通过 Webhook 发送至钉钉、企业微信、飞书 |
| 任务管理 | 创建和查询待办任务 |

### 两种运行模式

| 模式 | 说明 |
|------|------|
| **MCP Tools 模式** (默认) | 作为 MCP 服务器，供外部 MCP 客户端调用工具 |
| **Agent Brain 模式** | 启用内部 AI 工具自主规划，适合无人操控场景 |

## 3. 快速开始

### 3.1 本地开发环境

```bash
# 1. 进入源码目录
cd /Users/xiaokeji/Agent/02素材与源码/office-agent

# 2. 创建虚拟环境
python3 -m venv .venv

# 3. 激活虚拟环境
source .venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt

# 5. 复制环境配置文件
cp .env.example .env
# 编辑 .env 填入你的配置（见本文档第五章）

# 6. 初始化数据库
python -c "from app.database.db import init_db; init_db()"
```

### 3.2 启动服务

**启动 API 服务**
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**启动 MCP 服务（stdio 模式）**
```bash
python -m app.mcp_server.server
```

**启动 MCP 服务（HTTP 模式）**
```bash
MCP_TRANSPORT=streamable-http MCP_HOST=0.0.0.0 MCP_PORT=8010 python -m app.mcp_server.server
```

### 3.3 验证服务

```bash
# 验证 API 健康状态
curl http://localhost:8000/health

# 验证 MCP 服务
curl http://localhost:8010/health
```

## 4. Docker 部署

### 4.1 构建镜像

```bash
cd /Users/xiaokeji/Agent/02素材与源码/office-agent

# 构建镜像
docker build -t office-agent:latest .
```

### 4.2 docker-compose 部署

项目已提供 `docker-compose.yml`，支持两种服务：

| 服务 | 说明 | 端口 |
|------|------|------|
| mcp | MCP 工具服务器（默认启动） | 8010 |
| api | FastAPI REST 服务 | 8000 |

**启动 MCP 服务**
```bash
docker compose up -d mcp
```

**同时启动 MCP + API**
```bash
docker compose --profile api up -d
```

**启动 API 服务（单独）**
```bash
docker compose --profile api up -d api
```

### 4.3 查看服务状态

```bash
# 查看运行中的容器
docker compose ps

# 查看日志
docker compose logs -f mcp
docker compose logs -f api

# 停止服务
docker compose down
```

### 4.4 自定义 docker run 命令

**仅运行 API 服务**
```bash
docker run -d \
  --name office-agent-api \
  -p 8000:8000 \
  -e HOST=0.0.0.0 \
  -e PORT=8000 \
  -e APP_ENV=production \
  -v $(pwd)/data:/app/data \
  office-agent:latest \
  uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**仅运行 MCP 服务（HTTP）**
```bash
docker run -d \
  --name office-agent-mcp \
  -p 8010:8010 \
  -e MCP_TRANSPORT=streamable-http \
  -e MCP_HOST=0.0.0.0 \
  -e MCP_PORT=8010 \
  -e APP_ENV=production \
  -v $(pwd)/data:/app/data \
  office-agent:latest \
  python -m app.mcp_server.server
```

**运行 MCP 服务（stdio）**
```bash
docker run -d \
  --name office-agent-mcp-stdio \
  -e MCP_TRANSPORT=stdio \
  -e APP_ENV=production \
  -v $(pwd)/data:/app/data \
  office-agent:latest \
  python -m app.mcp_server.server
```

## 5. 配置文件说明

复制 `.env.example` 为 `.env`，然后根据实际情况修改。

### 5.1 完整配置示例

```bash
# ===== 应用基础配置 =====
APP_NAME=office-agent
APP_ENV=production
HOST=0.0.0.0
PORT=8000
PUBLIC_BASE_URL=https://your-domain.com

# ===== 数据库配置 =====
DATABASE_PATH=data/office_agent.db

# ===== MCP 服务配置 =====
# 传输模式：stdio | sse | streamable-http
MCP_TRANSPORT=streamable-http
MCP_HOST=0.0.0.0
MCP_PORT=8010
MCP_STREAMABLE_HTTP_PATH=/mcp
MCP_SSE_PATH=/sse
MCP_MESSAGE_PATH=/messages/
MCP_STATELESS_HTTP=false

# ===== Agent Brain 模式（可选） =====
# true = 启用自主 Agent 模式，需要配置 DeepSeek 或比赛允许的 AI 工具
# false = MCP 工具服务模式（默认）
ENABLE_AGENT_BRAIN=false

# DeepSeek API 配置（ENABLE_AGENT_BRAIN=true 时需要）
DEEPSEEK_API_KEY=your-deepseek-api-key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# 或者使用其他比赛允许的 AI 工具
# LLM_API_KEY=your-api-key
# LLM_BASE_URL=https://api.example.com/v1
# LLM_MODEL=your-model

# ===== 钉钉开放平台配置（可选）=====
DINGTALK_APP_KEY=your-dingtalk-app-key
DINGTALK_APP_SECRET=your-dingtalk-app-secret
# DINGTALK_API_BASE_URL=https://api.dingtalk.com

# ===== SMTP 邮件配置（可选） =====
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USERNAME=your-email@qq.com
SMTP_PASSWORD=your-auth-code
SMTP_TIMEOUT=30
SMTP_USE_TLS=true
SMTP_USE_STARTTLS=false

# Gmail 配置示例：
# SMTP_HOST=smtp.gmail.com
# SMTP_PORT=587
# SMTP_USE_TLS=false
# SMTP_USE_STARTTLS=true

# ===== 跨平台通知 Webhook（可选） =====
# JSON 格式，键为 channel_id，值包含 platform、name、webhook_url
NOTICE_CHANNELS={"dingding_dev_group":{"platform":"dingtalk","name":"钉钉研发群","webhook_url":"https://oapi.dingtalk.com/robot/send?access_token=xxx","secret":""},"wechat_project_group":{"platform":"wecom","name":"企业微信项目群","webhook_url":"https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx"}}
```

### 5.2 配置项详解

#### HOST / PORT
API 服务监听地址和端口。生产环境应设为 `0.0.0.0` 以允许外部访问。

#### MCP_TRANSPORT
MCP 服务传输模式：
- `stdio`: 同机器进程间通信，如连接 AstrBot MCP 等本地 MCP 客户端
- `sse`: Server-Sent Events，配合 HTTP 长连接
- `streamable-http`: 推荐用于网络通信，跨机器调用 MCP 工具

#### PUBLIC_BASE_URL
公网访问地址。当外部客户端（如钉钉机器人回调）需要访问本服务时填写。如果为空，则使用 `http://{HOST}:{PORT}`。

#### ENABLE_AGENT_BRAIN
- `false`（默认）: MCP 工具模式，作为工具服务器供外部客户端调用
- `true`: 启用内部 AI 工具自主规划和决策

#### SMTP 配置说明
| 邮件服务 | SMTP_HOST | 端口 | TLS | STARTTLS |
|---------|-----------|------|-----|----------|
| QQ 邮箱 | smtp.qq.com | 465 | true | false |
| QQ 企业邮箱 | smtp.exmail.qq.com | 465 | true | false |
| Gmail | smtp.gmail.com | 587 | false | true |
| Outlook | smtp.office365.com | 587 | false | true |

#### NOTICE_CHANNELS 格式
```json
{
  "channel_id_1": {
    "platform": "dingtalk",
    "name": "钉钉群名称",
    "webhook_url": "https://oapi.dingtalk.com/robot/send?access_token=xxx",
    "secret": "签名密钥（可选）"
  },
  "channel_id_2": {
    "platform": "wecom",
    "name": "企业微信群名称",
    "webhook_url": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx"
  }
}
```

## 7. MCP 服务调用

### 7.1 MCP Stdio 模式

适用于同机器上的 MCP 客户端（如 AstrBot MCP）。

**配置 JSON 示例**

```json
{
  "mcpServers": {
    "office-agent": {
      "command": "python",
      "args": ["-m", "app.mcp_server.server"],
      "env": {
        "PYTHONPATH": "/path/to/office-agent",
        "DATABASE_PATH": "/path/to/office-agent/data/office_agent.db"
      }
    }
  }
}
```

```json

  "mcpServers": {
```

## 8. MCP 工具详解

所有 MCP 工具均通过 `app/mcp_server/tools/` 目录下的模块实现。

### 8.1 文档处理工具

#### summarize_document_tool
总结文档内容，返回摘要、文本片段和字符数。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file_path | string | 否 | 本地文件路径 |
| fallback_text | string | 否 | 当 file_path 为空时使用的文本 |

**返回**
```json
{
  "summary": "文档摘要内容",
  "text_snippets": ["关键段落1", "关键段落2"],
  "char_count": 1234
}
```

#### read_document_tool
读取并解析文档内容，支持 Word、Excel、PDF、纯文本。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file_path | string | 是 | 文件路径 |
| extract_tables | boolean | 否 | 是否提取表格（默认 true） |
| extract_images | boolean | 否 | 是否提取图片（默认 false） |

**返回**
```json
{
  "success": true,
  "content": "文档文本内容",
  "tables": [["表头1", "表头2"], ["数据1", "数据2"]],
  "images": []
}
```

#### create_document_tool
根据结构化内容生成文件。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file_type | string | 是 | 输出格式：docx / xlsx / pdf / txt |
| content | object | 是 | 文件内容结构 |
| output_path | string | 是 | 输出文件路径 |

**content 格式示例（Word）**
```json
{
  "title": "文档标题",
  "paragraphs": ["第一段内容", "第二段内容"],
  "tables": [["表头1", "表头2"], ["数据1", "数据2"]]
}
```

#### convert_document_tool
文档格式转换。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| input_path | string | 是 | 源文件路径 |
| output_path | string | 是 | 目标文件路径 |
| target_format | string | 是 | 目标格式：docx / xlsx / pdf / txt |

### 8.2 表格处理工具

#### process_table_data_tool
对二维表格数据进行筛选、排序、合并、透视和统计分析。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| table_data | array | 是 | 二维数组，第一行为表头 |
| operation | string | 是 | 操作类型：filter / sort / merge / pivot / analyze |
| options | object | 否 | 操作选项 |

**filter 操作**
```json
{
  "column": "负责人",
  "value": "张三",
  "operator": "equals"
}
```

**sort 操作**
```json
{
  "column": "截止日期",
  "order": "asc"
}
```

**analyze 操作**
```json
{
  "group_by": "状态",
  "aggregations": {"数量": "sum", "金额": "avg"}
}
```

### 8.3 待办任务工具

## 9. 集成指南

### 9.1 与 AstrBot 集成

AstrBot 是一个支持 MCP 的 Bot 框架，可将 Office Agent 作为 MCP 工具集成为其添加工具能力。

**配置步骤**

1. 将 `mcp-stdio.astrbot-docker.example.json` 复制为 `mcp.json`
2. 根据实际路径修改 `PYTHONPATH` 和 `DATABASE_PATH`
3. 重启 AstrBot 服务

**示例 mcp.json**
```json
{
  "mcpServers": {
    "office-agent": {
      "command": "python",
      "args": ["-m", "app.mcp_server.server"],
      "env": {
        "PYTHONPATH": "/AstrBot/data/office-agent",
        "DATABASE_PATH": "/AstrBot/data/office-agent/data/office_agent.db"
      }
    }
  }
}
```

### 9.2 与本地 MCP 客户端集成

1. 找到本地 MCP 客户端配置文件
2. 添加 Office Agent MCP 服务器配置
3. 重启本地 MCP 客户端

### 9.3 敏感操作确认机制

以下工具涉及敏感操作，必须 `confirmed=true` 才会真正执行：

| 工具 | 敏感操作 |
|------|---------|
| create_tasks_tool | 写入任务数据 |
| send_email_tool | 发送邮件 |
| send_platform_notice_tool | 发送群通知 |

如果 `confirmed=false`，工具返回预览信息，等待外部 LLM 确认后再调用。
