## 数据流

### 邮件发送流程

```
用户消息 (包含邮件地址)
    │
    ▼
Planner: 识别 send_email intent
    │
    ▼
extract_email_addresses(): 提取邮件地址
    │
    ▼
WriterAgent.send_email_notification():
  - 调用 MCPToolClient.call("send_email_notification")
    │
    ▼
EmailService.send_email_notification():
  - 格式化通知内容为HTML
  - 调用 send_email()
    │
    ▼
EmailService._send_via_smtp():
  - 建立SMTP连接
  - 发送邮件
    │
    ▼
返回结果 (success, message_id, error)
```

### Agent集成流程

```
meeting_todos_and_notice intent:

1. extract_todos → 提取待办
2. generate_notice → 生成通知
3. [需要确认]
4. [用户确认]
5. create_tasks → 创建任务
6. send_email → 发送邮件通知
```

## 环境变量

新增SMTP相关环境变量（在`.env`中配置）:

```env
# SMTP Server Configuration
SMTP_HOST=smtp.qq.com              # SMTP服务器地址
SMTP_PORT=465                      # SMTP端口
SMTP_USERNAME=your-email@qq.com    # 发件人邮箱
SMTP_PASSWORD=your-auth-code       # 授权码/密码
SMTP_TIMEOUT=30                    # 连接超时（秒）
SMTP_USE_TLS=true                  # 使用隐式TLS（端口465）
SMTP_USE_STARTTLS=false            # 使用STARTTLS（端口587）
```

## 主要设计决策

1. **独立服务设计**: 
   - EmailService完全独立，不依赖AstrBot
   - 可独立测试和部署

2. **异步处理**:
   - 使用async/await支持高并发
   - 不阻塞Agent主流程

3. **MCP工具标准化**:
   - 遵循MCP协议
   - 易于扩展和集成

4. **配置灵活性**:
   - 支持多种SMTP服务商
   - 环境变量驱动配置

5. **安全性**:
   - 密码不硬编码
   - 配置验证机制
   - 异常捕获和处理

6. **用户友好**:
   - 自动地址识别
   - 清晰的错误提示
   - 完整的文档
