# Office Agent 对外调用与复用指南

本项目建议采用“双入口”对外复用：

1. **Agent HTTP API**：面向 AstrBot、网页、小程序、业务系统、脚本调用。
2. **MCP Server**：面向支持 MCP 协议的客户端、AstrBot MCP、自研 Agent 等。

这样可以同时证明项目具备完整 Agent 流程能力和 MCP 工具标准化复用能力。

## 1. 推荐部署形态

### 比赛演示或本机开发

```text
本机/AstrBot 容器
├── FastAPI Agent: http://127.0.0.1:8000/chat
└── MCP stdio: python -m app.mcp_server.server
```

适合快速演示、同机调试和 AstrBot Docker 内部调用。

### 正式部署或多工具复用

```text
服务器
├── Office Agent API: http://server:8000/chat
├── Office Agent Capabilities: http://server:8000/capabilities
└── MCP Streamable HTTP: http://server:8010/mcp
```

推荐把 Agent API 和 MCP Server 都部署在服务器上：

- 外部业务系统调用 `/chat`，获得完整办公流程编排。
- 支持 MCP 的 AI 工具连接 `/mcp`，复用具体办公工具。
- AstrBot 仍可作为聊天平台入口，通过插件调用 `/chat`。

## 2. HTTP Agent API

HTTP API 适合普通系统调用，不要求调用方支持 MCP。

### 健康检查

```bash
curl http://127.0.0.1:8000/health
```

### 查看能力清单

```bash
curl http://127.0.0.1:8000/capabilities
```

### 调用 Agent 流程

```bash
curl -X POST http://127.0.0.1:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "platform": "external-tool",
    "user_id": "u001",
    "session_id": "external-demo-001",
    "content": "请把会议纪要整理成待办并生成通知：张三周五前完成需求文档，李四下周一提交测试计划。"
  }'
```

### 上传文件

```bash
curl -X POST http://127.0.0.1:8000/files/upload \
  -F "file=@/path/to/meeting.docx" \
  -F "platform=web" \
  -F "user_id=u001" \
  -F "session_id=s001"
```

返回 `file_path` 后，可在 `/chat` 请求中传入：

```json
{
  "platform": "external-tool",
  "user_id": "u001",
  "session_id": "s001",
  "content": "请读取这个文档并总结重点",
  "file": {
    "file_name": "meeting.docx",
    "file_path": "data/files/meeting.docx"
  }
}
```

## 3. MCP stdio 模式

stdio 适合同一台机器或同一个容器内的 MCP 客户端。

示例配置见：

```text
deploy/mcp-stdio.example.json
deploy/mcp-stdio.astrbot-docker.example.json
```

AstrBot Docker 内可以使用：

```json
{
  "command": "/usr/bin/env",
  "args": [
    "PYTHONPATH=/AstrBot/data/office-agent",
    "DATABASE_PATH=/AstrBot/data/office-agent/data/office_agent.db",
    "MCP_TRANSPORT=stdio",
    "/AstrBot/data/office-agent/.venv/bin/python",
    "-m",
    "app.mcp_server.server"
  ]
}
```

## 4. MCP Streamable HTTP 模式

Streamable HTTP 适合服务器部署和远程工具调用。

启动命令：

```bash
cd /AstrBot/data/office-agent
source .venv/bin/activate

PYTHONPATH=/AstrBot/data/office-agent \
DATABASE_PATH=/AstrBot/data/office-agent/data/office_agent.db \
MCP_TRANSPORT=streamable-http \
MCP_HOST=0.0.0.0 \
MCP_PORT=8010 \
python -m app.mcp_server.server
```

MCP 地址：

```text
http://服务器IP:8010/mcp
```

如果通过宝塔/Nginx 反向代理，可以参考：

```text
deploy/reverse-proxy.nginx.example.conf
```

## 5. 对外复用边界

| 入口 | 面向对象 | 推荐用途 |
|---|---|---|
| `/chat` | 普通业务系统、AstrBot 插件、网页、小程序 | 完整办公流程、上下文、多 Agent 协同、确认流程 |
| `/files/upload` | 普通业务系统、Web 前端 | 上传文档并登记会话文件 |
| `/capabilities` | 第三方系统、评审演示 | 查看系统能力、端点、MCP 工具清单 |
| MCP stdio | 同机 MCP 客户端 | 本地复用工具能力 |
| MCP Streamable HTTP | 远程 MCP 客户端 | 服务器化复用工具能力 |

## 6. Agent 级 MCP 调用

如果调用方希望直接使用完整办公 Agent，而不是逐个调用 `extract_todos_tool`、`generate_notice_tool` 等工具，可以使用：

```text
office_agent_chat_tool
```

参数示例：

```json
{
  "content": "请把会议纪要整理成待办并生成通知：张三周五前完成需求文档，李四下周一提交测试计划。",
  "user_id": "u001",
  "session_id": "mcp-demo-001",
  "platform": "mcp-client"
}
```

返回结果与 HTTP `/chat` 基本一致：

```json
{
  "session_id": "mcp-demo-001",
  "reply": "已提取出以下待办...",
  "intent": "meeting_todos_and_notice",
  "need_confirmation": true,
  "tool_results": []
}
```

如果返回 `need_confirmation=true`，MCP 客户端可以继续用同一个 `session_id` 调用：

```json
{
  "content": "确认",
  "user_id": "u001",
  "session_id": "mcp-demo-001",
  "platform": "mcp-client"
}
```

这样完整保留了 Agent 的上下文、确认流程和多角色协同。

## 7. 最终定位

本项目不是只能本地运行的聊天机器人，而是：

```text
服务器可部署的 Office Agent API
        +
可本地/远程复用的 MCP 工具服务器
        +
AstrBot 多平台聊天入口插件
```

推荐答辩表述：

> 系统采用“Agent API + MCP Server + AstrBot 插件”的分层设计。Agent API 负责完整办公流程编排，MCP Server 负责工具能力标准化复用，AstrBot 插件负责多平台消息入口。因此系统既可以本地部署演示，也可以服务器部署后被其他工具通过 HTTP API 或 MCP 协议调用。
