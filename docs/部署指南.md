# Office Agent 部署指南

## 架构概览

Office Agent 提供两种调用接口：

```
外部调用方                           Office Agent
                                   ┌─────────────────────────────┐
  HTTP POST /chat                  │  FastAPI (端口 8000)        │
  ────────────────────────────────►│  └── Agent 核心处理        │
                                   └─────────────────────────────┘

  MCP 客户端                         │  MCP Server (端口 8010)     │
  ────────────────────────────────►│  ├── office_agent_chat_tool │
  (streamable-http / stdio)        │  └── 基础工具集              │
                                   └─────────────────────────────┘
```

**两种部署模式：**
- **API 模式**：直接 HTTP 调用，适合第三方系统集成
- **MCP 模式**：作为 MCP 服务器被 AstrBot 或其他 MCP 客户端调用

## 一、服务器部署步骤

### 1. 上传代码

```bash
# 在服务器上克隆或上传项目
scp -r office-agent/ user@your-server:/AstrBot/data/office-agent
```

### 2. 创建虚拟环境并安装依赖

```bash
cd /AstrBot/data/office-agent
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
```

### 3. 配置环境变量

```bash
# 创建 .env 文件
cat > /AstrBot/data/office-agent/.env << 'EOF'
# 应用配置
APP_NAME=office-agent
APP_ENV=production

# DeepSeek API（可选，未配置时使用本地规则流程）
DEEPSEEK_API_KEY=your-api-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# 数据库
DATABASE_PATH=/AstrBot/data/office-agent/data/office_agent.db

# 公共访问地址（部署在反向代理后时填写）
PUBLIC_BASE_URL=http://your-server.example.com:8000

# SMTP 邮件配置（可选）
SMTP_HOST=smtp.example.com
SMTP_PORT=465
SMTP_USERNAME=your-email@example.com
SMTP_PASSWORD=your-password
SMTP_USE_TLS=true

# 文件上传
MAX_FILE_SIZE_MB=20
UPLOAD_DIR=data/files

# MCP 服务配置
MCP_TRANSPORT=streamable-http
MCP_HOST=0.0.0.0
MCP_PORT=8010
MCP_STREAMABLE_HTTP_PATH=/mcp
MCP_STATELESS_HTTP=false
EOF
```

### 4. 初始化数据库

```bash
./.venv/bin/python -c "from app.database.db import init_db; init_db()"
```

## 二、API 模式部署（HTTP 调用）

API 模式适合直接通过 HTTP 调用 Agent，适合任何能发送 HTTP 请求的系统。

### 启动服务

**方式 A：直接运行（开发/测试用）**
```bash
cd /AstrBot/data/office-agent
./.venv/bin/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**方式 B：systemd 服务（生产推荐）**

```bash
# 复制服务文件
sudo cp deploy/office-agent-api.service.example /etc/systemd/system/office-agent-api.service

# 编辑服务文件，修正路径
sudo vim /etc/systemd/system/office-agent-api.service
# 确保 WorkingDirectory 和 ExecStart 路径正确

# 启动并设置开机自启
sudo systemctl daemon-reload
sudo systemctl enable office-agent-api
sudo systemctl start office-agent-api

# 验证状态
sudo systemctl status office-agent-api
```

### 调用示例

```bash
# 直接 curl 调用
curl -sS -X POST http://127.0.0.1:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "platform": "external",
    "user_id": "u001",
    "session_id": "session-001",
    "content": "请把会议纪要整理成待办：张三周五前完成需求文档，李四下周一提交测试计划。"
  }'
```

**Python 调用：**
```python
import httpx

response = httpx.post("http://your-server:8000/chat", json={
    "platform": "external",
    "user_id": "u001",
    "session_id": "session-001",
    "content": "请总结这段话：今天完成了项目评审。",
})
print(response.json())
```

**响应格式：**
```json
{
  "session_id": "session-001",
  "reply": "已提取出以下待办：\n1. 完成项目评审（负责人：待补充，截止时间：待补充）",
  "intent": "extract_todos",
  "need_confirmation": false,
  "tool_results": [...]
}
```

### Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name your-server.example.com;

    client_max_body_size 30m;

    location /agent/ {
        proxy_pass http://127.0.0.1:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用 HTTPS：
```bash
sudo certbot --nginx -d your-server.example.com
```

## 三、MCP 模式部署（远程 MCP 调用）

MCP 模式将 Agent 作为 MCP 服务器， AstrBot 或其他 MCP 客户端通过 MCP 协议调用。

### 启动 MCP 服务

**方式 A：systemd 服务（生产推荐）**

```bash
# 复制服务文件
sudo cp deploy/office-agent-mcp-http.service.example /etc/systemd/system/office-agent-mcp-http.service

# 编辑服务文件，确认路径
sudo vim /etc/systemd/system/office-agent-mcp-http.service

# 启动
sudo systemctl daemon-reload
sudo systemctl enable office-agent-mcp-http
sudo systemctl start office-agent-mcp-http
```

**方式 B：直接运行**
```bash
cd /AstrBot/data/office-agent
MCP_TRANSPORT=streamable-http \
MCP_HOST=0.0.0.0 \
MCP_PORT=8010 \
MCP_STREAMABLE_HTTP_PATH=/mcp \
./.venv/bin/python -m app.mcp_server.server
```

### 调用 MCP 工具

MCP 工具列表：

| 工具名 | 说明 | 关键参数 |
|--------|------|---------|
| `office_agent_chat_tool` | 调用完整 Agent 流程 | `content`, `user_id`, `session_id` |
| `summarize_document_tool` | 文档摘要 | `file_path`, `fallback_text` |
| `read_document_tool` | 读取文档内容 | `file_path`, `extract_tables`, `extract_images` |
| `create_document_tool` | 创建文档 | `file_type`, `content`, `output_path` |
| `convert_document_tool` | 文档格式转换 | `input_path`, `output_path`, `target_format` |
| `process_table_data_tool` | 处理表格数据 | `table_data`, `operation`, `options` |
| `extract_todos_tool` | 提取待办事项 | `text` |
| `create_tasks_tool` | 创建任务 | `todos`, `source_platform`, `source_user_id` |
| `query_task_status_tool` | 查询任务状态 | `user_id` |
| `generate_notice_tool` | 生成通知 | `todos` |
| `generate_email_tool` | 生成邮件草稿 | `content` |
| `send_email_tool` | 发送邮件 | `to_addresses`, `subject`, `body` |
| `send_email_notification_tool` | 发送通知邮件 | `to_address`, `notice_content` |
| `validate_email_config_tool` | 验证邮件配置 | 无 |

### AstrBot 集成配置

**方式 A：stdio 模式（本地进程调用）**

在 AstrBot 管理界面或配置文件中添加 MCP 工具：

```json
{
  "mcpServers": {
    "office-agent": {
      "command": "/AstrBot/data/office-agent/.venv/bin/python",
      "args": ["-m", "app.mcp_server.server"],
      "env": {
        "PYTHONPATH": "/AstrBot/data/office-agent",
        "DATABASE_PATH": "/AstrBot/data/office-agent/data/office_agent.db",
        "MCP_TRANSPORT": "stdio"
      }
    }
  }
}
```

**方式 B：streamable-http 模式（Docker 部署的 AstrBot）**

```json
{
  "command": "/usr/bin/env",
  "args": [
    "PYTHONPATH=/AstrBot/data/office-agent",
    "DATABASE_PATH=/AstrBot/data/office-agent/data/office_agent.db",
    "MCP_TRANSPORT=streamable-http",
    "/AstrBot/data/office-agent/.venv/bin/python",
    "-m",
    "app.mcp_server.server"
  ]
}
```

**AstrBot Docker 环境变量方式：**
```bash
# 在 AstrBot 的 docker-compose 或环境变量中配置
PYTHONPATH=/AstrBot/data/office-agent
MCP_TRANSPORT=streamable-http
MCP_HOST=office-agent-mcp  # MCP 服务的主机名或 IP
MCP_PORT=8010
```

### 远程调用 office_agent_chat_tool

```bash
# 发送 chat 消息给 Agent
curl -sS -X POST http://your-server:8010/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "office_agent_chat_tool",
      "arguments": {

        "user_id": "astrbot_user",
        "session_id": "astrbot-session-001"
      }
    },
    "id": 1
  }'
```

## 四、Nginx 反向代理 MCP 流量

如果需要通过域名访问 MCP 服务：

```nginx
server {
    listen 80;
    server_name your-server.example.com;

    location /mcp/ {
        proxy_pass http://127.0.0.1:8010/mcp/;
        proxy_http_version 1.1;
        proxy_buffering off;
        proxy_cache off;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 六、常见问题

### 1. MCP 连接失败

```bash
# 检查端口是否监听
ss -tlnp | grep 8010

# 检查服务日志
sudo journalctl -u office-agent-mcp-http -n 50

# 测试 MCP 端点
curl http://127.0.0.1:8010/mcp
```

### 2. 工具调用返回 None 或失败

检查 `.env` 中 `PUBLIC_BASE_URL` 是否正确配置。如果 MCP 服务在反向代理后，外部客户端需要通过公网地址访问。

### 3. 邮件发送失败

```bash
# 验证 SMTP 配置
./.venv/bin/python -c "
import asyncio
from app.mcp_client.client import MCPToolClient
async def test():
    client = MCPToolClient()
    result = await client.call('validate_email_config', {})
    print(result)
asyncio.run(test())
"
```

### 4. AI 工具调用失败

确保 `DEEPSEEK_API_KEY` 正确配置。未配置时系统自动降级到本地规则流程，但功能受限。
