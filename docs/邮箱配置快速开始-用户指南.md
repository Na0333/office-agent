# 邮箱配置快速开始 - 用户指南

## 🚀 一键启动（推荐）

如果你是第一次使用 Office Agent，最简单的方式是运行一键初始化工具：

```bash
python init.py
```

这个工具提供了一个友好的菜单界面，让你可以：

```
1️⃣  交互式邮箱配置   - 按步骤配置你的邮箱
2️⃣  配置诊断          - 检查现有配置是否正确
3️⃣  快速测试          - 快速测试邮件功能
4️⃣  查看文档          - 查看详细使用文档
5️⃣  启动 Agent        - 启动 Office Agent 服务
0️⃣  退出              - 退出程序
```

## 📝 分步指南

### 方法1：使用交互式配置向导（最推荐 ⭐⭐⭐⭐⭐）

```bash
python setup_email.py
```

**步骤：**

1. **选择邮箱提供商**
   - QQ 邮箱
   - Gmail
   - Outlook
   - 腾讯企业邮
   - 其他 SMTP 服务

2. **输入邮箱地址**
   ```
   user@example.com
   ```

3. **输入授权码**
   - 系统会自动显示如何获取授权码
   - 不同邮箱获取方法不同

4. **确认配置**
   - 查看配置摘要
   - 确认无误后保存

5. **测试连接**
   - 系统会自动验证 SMTP 连接
   - 可选：发送测试邮件

### 方法2：手动编辑 .env 文件

如果你了解 SMTP 配置，可以直接编辑配置文件：

```bash
# 复制示例文件
cp .env.example .env

# 编辑 .env
nano .env  # 或使用你喜欢的编辑器
```

**QQ 邮箱配置示例：**

```env
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USERNAME=your-email@qq.com
SMTP_PASSWORD=your-auth-code
SMTP_TIMEOUT=30
SMTP_USE_TLS=true
SMTP_USE_STARTTLS=false
```

**Gmail 配置示例：**

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_TIMEOUT=30
SMTP_USE_TLS=false
SMTP_USE_STARTTLS=true
```

### 方法3：配置诊断和修复

## 🔧 常见邮箱配置

### QQ 邮箱

**获取授权码：**
1. 登录 QQ 邮箱
2. 进入 "设置" → "账户"
3. 找到 "POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
4. 点击 "生成授权码"
5. 将授权码复制到 `SMTP_PASSWORD`

**配置参数：**
```env
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USE_TLS=true
SMTP_USE_STARTTLS=false
```

### Gmail

**获取应用密码：**
1. 访问 myaccount.google.com
2. 进入 "安全" 标签
3. 启用 "两步验证"
4. 生成 "应用专用密码"
5. 将密码复制到 `SMTP_PASSWORD`

**配置参数：**
```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USE_TLS=false
SMTP_USE_STARTTLS=true
```

### Outlook / Hotmail

**配置参数：**
```env
SMTP_HOST=smtp.office365.com
SMTP_PORT=587
SMTP_USE_TLS=false
SMTP_USE_STARTTLS=true
```

### 腾讯企业邮

**配置参数：**
```env
SMTP_HOST=smtp.exmail.qq.com
SMTP_PORT=465
SMTP_USE_TLS=true
SMTP_USE_STARTTLS=false
```

## 🐛 故障排除

### 问题1：连接超时

**症状：** SMTP 连接超时

**原因：**
- SMTP 服务器地址错误
- 防火墙阻止连接
- 网络连接问题

**解决方案：**
1. 检查 SMTP_HOST 是否正确
2. 检查 SMTP_PORT 是否正确（465 or 587）
3. 检查网络连接
4. 检查防火墙设置

### 问题2：认证失败

**症状：** 邮箱或密码错误

**原因：**
- SMTP_USERNAME 或 SMTP_PASSWORD 错误
- 授权码已过期

**解决方案：**
1. 确认邮箱地址正确
2. 重新获取授权码（某些邮箱定期过期）
3. 检查是否使用了错误的密码（应使用授权码）

### 问题3：TLS 错误

**症状：** TLS 握手失败

**原因：**
- TLS 配置错误
- 端口与 TLS 设置不匹配

**解决方案：**
- 端口 465 用 `SMTP_USE_TLS=true`
- 端口 587 用 `SMTP_USE_STARTTLS=true`

### 问题4：邮件未收到

**症状：** 邮件发送成功但未收到

**原因：**
- 邮件进入垃圾邮件文件夹
- 邮件服务商限制

**解决方案：**
1. 检查垃圾邮件文件夹
2. 添加发件人到联系人
3. 减少发送频率
4. 联系邮件服务商

## 💡 使用提示

- 📧 **邮件配置是可选的** - 没有邮件配置也能使用待办、通知等功能
